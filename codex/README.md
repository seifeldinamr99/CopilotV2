# Codex Working Set

This folder is the durable, small set of files we keep updated so future Codex sessions don’t need to re-scan the entire repo.

Use these as the source of truth:
- `codex/CONTEXT.md` — current architecture + invariants
- `codex/STATUS.md` — what’s done / next / blockers
- `codex/API_CONTRACT.md` — endpoint contract (request/response/auth)
- `codex/DB_SCHEMA.md` — Prisma schema notes + invariants
- `codex/WORKFLOW.md` — how we use Codex (PR-style)
- `codex/REVIEW_CHECKLIST.md` — “QA engineer” checklist per change
- `codex/SESSION_HANDOFF.md` — what to paste into a new session
